# 如何测试 Tiaoom 游戏开发 Claude Skill

本文档介绍如何测试和使用 `tiaoom-game-development.md` 这个 Claude Skill。

## 测试方法

### 方法一：在 Claude 对话中使用（推荐）

1. **打开 Claude 对话**
   - 访问 https://claude.ai
   - 开始一个新对话

2. **上传 Skill 文件**
   - 将 `tiaoom-game-development.md` 文件上传到对话中
   - 或者直接复制文件内容粘贴到对话框

3. **测试场景示例**

   **场景 1：询问如何开始开发游戏**
   ```
   我想为 Tiaoom 平台开发一个井字棋游戏，请问我需要做什么？
   ```
   
   **场景 2：询问具体功能实现**
   ```
   如何在游戏中实现倒计时功能？玩家超时后自动跳过回合。
   ```
   
   **场景 3：请求代码示例**
   ```
   帮我写一个石头剪刀布游戏的后端代码框架。
   ```
   
   **场景 4：调试问题咨询**
   ```
   我的游戏在玩家断线重连后状态丢失了，应该如何解决？
   ```
   
   **场景 5：询问最佳实践**
   ```
   在 Tiaoom 游戏开发中，如何正确处理游戏结束时的积分奖励？
   ```

4. **验证 Claude 回答**
   
   Claude 应该能够：
   - ✅ 基于 Skill 中的知识回答问题
   - ✅ 提供符合 Tiaoom 规范的代码示例
   - ✅ 引用 Skill 中的具体章节和方法
   - ✅ 给出完整的前后端实现方案
   - ✅ 解释游戏开发的最佳实践

### 方法二：实际开发测试

1. **克隆 Tiaoom 项目**
   ```bash
   git clone https://github.com/FishPiOffical/tiaoom.git
   cd tiaoom/game
   npm install
   ```

2. **使用 Skill 辅助开发新游戏**
   - 将 Skill 文件提供给 Claude
   - 请求 Claude 帮助你开发一个简单游戏（如猜数字）
   - 按照 Claude 的指导实现代码
   - 运行测试验证功能

3. **验证开发流程**
   ```bash
   # 启动开发服务器
   npm run dev
   
   # 前端访问 http://localhost:5174
   # 后端运行在 http://127.0.0.1:27015
   ```

4. **检查生成的代码**
   - ✅ 后端文件是否在 `backend/src/games/` 目录
   - ✅ 前端组件是否在 `frontend/src/components/games/` 目录
   - ✅ 代码是否符合 Tiaoom 的命名规范
   - ✅ 游戏是否能正常创建房间和运行
   - ✅ 状态同步是否正常工作

### 方法三：Skill 内容验证

1. **检查 Skill 完整性**
   ```bash
   # 查看 Skill 文件
   cat tiaoom-game-development.md
   
   # 检查行数（应该有 700+ 行）
   wc -l tiaoom-game-development.md
   ```

2. **验证关键章节是否存在**
   - [ ] 后端开发规范
   - [ ] 前端开发规范
   - [ ] GameRoom 核心方法说明
   - [ ] 倒计时实现
   - [ ] 游戏数据管理
   - [ ] 游戏回放
   - [ ] 完整示例（抢数字游戏）
   - [ ] 最佳实践
   - [ ] 常见问题解答

3. **搜索特定内容**
   ```bash
   # 检查是否包含 GameRoom 相关内容
   grep -i "GameRoom" tiaoom-game-development.md
   
   # 检查是否包含完整示例
   grep -i "完整示例" tiaoom-game-development.md
   
   # 检查是否包含倒计时说明
   grep -i "倒计时" tiaoom-game-development.md
   ```

## 示例对话

### 示例 1：创建新游戏

**用户：**
```
我想创建一个猜数字游戏，电脑随机生成一个 1-100 的数字，玩家轮流猜，
谁猜中谁获胜。请帮我实现。
```

**Claude 应该回答：**
- 提供完整的后端代码（继承 GameRoom）
- 提供完整的前端 Vue 组件
- 说明如何配置游戏（minSize, maxSize 等）
- 解释游戏流程和状态管理
- 提供测试运行的步骤

### 示例 2：功能实现

**用户：**
```
如何在游戏中添加 30 秒倒计时？超时自动跳过当前玩家。
```

**Claude 应该回答：**
- 后端使用 `startTimer()` 方法
- 实现 `restoreTimer()` 恢复逻辑
- 前端监听 `countdown` 指令
- 显示倒计时 UI
- 提供完整代码示例

### 示例 3：问题排查

**用户：**
```
我的游戏结束后玩家无法离开房间，怎么办？
```

**Claude 应该回答：**
- 检查是否调用了 `this.room.end()`
- 说明在哪里调用这个方法
- 提供代码示例
